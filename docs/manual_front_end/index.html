HTML

Na Internit, utilizamos o HTML5. Abaixo existem algumas dicas para deixar o seu código mais organizado:

  • Elementos aninhados devem estar indentados uma vez;
  • Sempre use aspas duplas;
  • Utilize sempre o DOCTYPE do HTML5;
  • Utilize sempre o charset UTF-8;
  • Nunca inclua uma barra invertida nos elementos únicos (exemplo: img ou hr). No HTML5 isso não é mais necessário;
  • Utilize sempre o alt nas imagens
                        <!DOCTYPE html>
                        <html>
                            <head>
                                <title>Título da Página</title>
                            </head>

                            <body>
                                <img src="imagens/logo.png" alt="Meu Logo">
                                <h1 class="destaque">Olá Mundo!</h1>
                            </body>
                        </html>
  • De acordo com as especificações do HTML5, não há necessidade de especificar um atributo type quando incluímos arquivos de CSS e JavaScript como text/css e text/javascript;
                        <-- No HTML 4 -->
                        <link type="text/css" rel="stylesheet" href="main.css">
                        <script type="text/javascript" src="geral.js">


                        <-- No HTML 5 -->
                        <link rel="stylesheet" href="main.css">
                        <script src="geral.js">
  • Esforce-se para manter o padrão e a semântica do HTML, mas não às custas da praticidade;
  • Use o mínimo de código com a menor complexidade possível;
  • Sempre que possível, evite elementos pais supérfluos quando escrever HTML;
                        <-- Não muito bom -->
                        <span class="avatar">
                            <img src="...">
                        </span>


                        <-- Melhor -->
                        <img class="avatar" src="...">

CSS

Abaixo algumas dicas para deixar seu código CSS mais organizado:

  • Quando agrupar seletores, deixe seletores individuais em uma linha;
  • Inclua um espaço antes de abrir um {};
  • Coloque o fechamento do {} em uma nova linha;
  • Inclua um espaço depois dos dois pontos em cada declaração de propriedade;
  • Cada declaração deve estar em sua própria linha para encontrarmos melhor os erros;
  • Feche todas as declarações com ponto-vírgula;
  • Também não coloque valores iniciados com zero (ex., .5 em vez de 0.5);
  • Valores separados por vírgula devem incluir um espaço logo depois da vírgula;
  • Use cores em hexadecimal abrevidas quando puder, exemplo: #fff em vez de #fff;
  • Coloque aspas duplas nos valores em seletores com atributos, exemplo: input[type="text"];
  • Evite especificar unidades quando os valores forem 0, exemplo: margin: 0 em vez de margin: 0px;
  • Só utilize position absolute quando for realmente necessário
                        /* RUIM */
                       .texto, .conteudo, input[type=text] {
                            padding:15px;
                            margin:0px 0px 15px;
                            background-color:rgba(0, 0, 0, 0.5);
                            box-shadow:0 1px 2px #ccc,inset 0 1px 0 #ffffff
                        }


                        /* BOM */
                       .texto,
                       .conteudo,
                       input[type="text"] {
                            padding: 15px;
                            margin: 0 0 15px;
                            background-color:rgba(0,0,0,.5);
                            box-shadow:0 1px 2px #ccc, inset 0 1px 0 #fff;
                        }
  • base.css - Como o nome diz, esse é o CSS base do site. Ele foi baseado no bootstrap e é nele que ficam os estilos das grids, reset, plugins do bootstrap, etc. Recomendável não mexer nesse arquivo.
  • combobox.css - É o css do plugin que utilizamos para estilar o select, radio e checkbox
  • formularios.css - CSS utilizado para estilo dos formulários
  • main.css - CSS principal com os estilos do header, home e footer
  • owl.carousel.css - CSS do plugin de slider
  • prettyPhoto.css - CSS do plugin de ampliação de imagens
  • responsive.css - CSS utilizado para sites responsivos. Se o projeto em que você está trabalhando não for responsivo, tire a chamada dele no layout.twig.html
  • secundarias.css - Utilizado para estilo das páginas internas

Entendendo o Twig

A primeira coisa que se deve fazer quando for recortar um novo site, é baixar a versão do site base.

Como iniciar um projeto:

  • Com o terminal aberto, escolha a pasta onde o projeto será instalado.
  • Execute o comando abaixo para fazer uma cópia do site base para o diretório escolhido: 
    composer create-project --repository-url=http://packs.lab58.internit.br internit/site-base . dev-master
  • Depois do comando acima, peça para algum programador criar o banco de dados. O sistema vai pedir algumas configurações e você preencherá 4 delas. As outras, basta deixar em branco e aperta enter.

    database_host<localhost> - db01.lab58.internit.br

    database_name<site-base> - nome do banco de dados (o mesmo nome que o programador criou)

    database_user<root> - labuser

    database_password<root> - labpass

  • Depois de ter o banco criado, execute o comando abaixo para criar as tabelas: 
    php app\console doctrine:schema:update --force
  • Para criar uma conta de admin, utilize: 
    php app\console admin:create:user
  • Depois, escolha uma senha para o admin e aperte enter. Por padrão, utilizamos IN@abc
  • Execute o comando abaixo para criar o menu: 
    php app\console admin:menu:root
  • E por último, execute o comando para baixar os arquivos do tupi: 
    composer update

Você só vai precisa mexer na pasta "src". Como o nome diz, é nessa pasta que fica a source do site, ou seja, todo o código fonte principal.

Os que estão em destaque, são os que você precisa conhecer:

  • EventListener: O único arquivo que você precisará mexer nessa pasta, é o PageChangeListener.php. É nele que você define os templates que aparecerão no admin.

Dentro da pasta Resource, existem duas páginas que precisa conhecer: public e views.

  • Public: É nela em que ficarão os assets, ou seja, o css, o js e os arquivos de media (img, pdf, swf, etc).
  • Views: Se encontram os arquivos HTML do site. São divididos em dois:
    • Default: É nessa pasta que ficam as suas páginas HTML.
    • Template: É nessa pasta que ficam os arquivos base: header, footer, etc. Arquivos que serão chamados praticamente em todas as páginas do site.


Para acessar o site, utilize a seguinte url: caminho_de_onde_instalou_o_site/web

E para acessar o admin: caminho_de_onde_instalou_o_site/web/admin

Controle de versão SVN

Depois que você criou um novo projeto e modificou ele, antes de ir embora, lembre-se sempre de commitar.

Caso o projeto ainda não esteja no controle de versão, peça para Macedo criar um novo diretório. Depois que ele foi criado, vá no Eclipse e clique com o botão direito no projeto > Team > Share Project...

Escolha a opção SVN

Selecione "Create a new repository location" e na próxima tela, jogue o caminho do novo diretório que foi criado e depois clique em "Finish".

Para commitar seus arquivos, clique com o botão direito no projeto e selecione "Commit...". Atenção: Caso o botão de "Update" esteja habilitado, selecione ele antes de commitar. Caso os arquivos da sua máquina estejam errado e você queira substituí-los pelo do controle de versão, basta clicar em "Revert...".

A tela abaixo irá aparecer quando você der o commit, coloquei o número da tarefa e diga o que você alterou.
